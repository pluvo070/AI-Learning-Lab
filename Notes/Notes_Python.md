# Python

## Python 基础

### LEGB 作用域

- **Python 是动态语言**

  - 在 Java 中，如果你写 `count = 10`，编译器会先检查这是否是一个已声明的局部变量，如果不是，它会去检查这是否是一个类成员变量。**因为 Java 必须先声明类型**，所以编译器能通过声明明确知道你的意图。

  - **Python 是动态语言，不需要声明**。 <u>赋值 = 默认新建本地变量</u>。如果 Python 允许 “找不到局部就自动改外部”，会发生意外覆盖：

    ```py
    x = 10
    def test():
        x = 5  	# 新建一个局部变量 x 并赋值为 5
        		# 如果Python自动找外部，这行就会把全局的x改了
    ```

    Python 的逻辑是：<u>在函数内部，只要出现了赋值语句（如 `x = ...`），Python 就默认 x 是局部变量。</u> 如果你想动外面的，你必须“显式授权”（用 `global` 或 `nonlocal`）。

- **LEGB 法则**：Python 找变量不是“乱找”，而是严格遵循 LEGB 路径，且只能 **由内向外**：

  1. **L (Local)**：当前函数内部
  2. **E (Enclosing)**：外部嵌套函数（即 `nonlocal` 所在的层级）
  3. **G (Global)**：当前模块（文件）的最外层
  4. **B (Built-in)**：Python 内置的函数（如 `print`, `len`）

- **显式声明变量权限**：

  - `nonlocal`：从当前层往外找，直到找到第一个匹配的变量为止，但它只在“嵌套函数”里找，不会找全局（Global）。

    > 如果所有嵌套层都没有它就会报错，但它不会越界去打扰全局变量。如果你想动全局的，必须用 `global`。

  - `global`：声明这个变量是全局变量

  - `global` 和 `nonlocal` **不是类型声明，而是“权限声明”**。

    - 因此 <u>不能在定义时声明</u>：不能写 `global x = 10`（报语法错误）。
    - 必须 <u>在函数体内声明</u>：通常写在函数的第一行。

- **在函数中修改外部变量**：使用列表字典、使用 `nonlocal` 声明、使用 `global` 声明

  - 如果在函数内部修改函数外部定义的简单变量，会有“权限问题”

    ```py
    count = 0  # 外部变量
    
    def log_tool_calls(request, handler):
        count += 1  # 报错：UnboundLocalError
    ```

    > 当 Python 解释器在函数内部看到 `call_count += 1`时，它会认为 `call_count` 是一个**局部变量**。
    >
    > 当它尝试计算 `call_count + 1` 时，它发现局部变量 `call_count` 还没有被赋值，于是就崩溃了。

  - **使用容器变量**（`list`, `dict`）：在函数里可以直接**改**里面的内容。

    ```py
    count = [0]  # 外部变量
    
    def log_tool_calls(request, handler):
        count[0] += 1 
    ```

  - **使用 `nonlocal` 关键字**：声明这个变量在外层但非全局，告诉 Python 去上几层找这个变量

    ```py
    count = 0  # 外部变量
    
    def log_tool_calls(request, handler):
        nonlocal count  # 声明要修改外部变量
        count += 1
    ```

  - **使用 `global` 关键字**：声明这个变量在全局，告诉 Python 去全局变量找

    ```py
    count = 0 # 外部变量
    
    def log_tool_calls(request, handler):
        global count  # 声明要修改全局的那个count
        count += 1
    ```

- **类（Class）里的成员函数**

  - 不需要预先声明成员变量，在初始化函数中赋值即定义

  - 成员变量必须写成 `self.x`

    ```py
    class MyAgent:
        def __init__(self):
            self.count = 0  # 定义了成员变量count
    
        def update(self):
            count = 10      # 函数内的局部变量
            self.count = 5  # 只有带 self 的才是成员变量
    ```



## 张量 Tensor

- **==张量== (Tensor)** ： 多维数组。矩阵是张量的一种特例（2 维张量）。在 PyTorch 里，不管几维，统一都叫 `Tensor`。

  - **0 维张量**：标量 (Scalar)。就是一个数字，比如 `5`。
  - **1 维张量**：向量 (Vector)。Java 里的 `int[]` 或 `List<Integer>`。
  - **2 维张量**：矩阵 (Matrix)。Java 里的 `int[][]`。
  - **3 维及以上**：就叫张量。比如一张彩色图片，就是 `[高度, 宽度, 3个颜色通道]` 的 3 维张量。

- **==维度==**：用 **`dim`** 或 **`axis`** 表示，索引从 **0** 开始。

  如果有一个形状为 `[2, 3, 4]` 的张量：

  - **`dim=0`**：第一层包装，里面有 2 个元素（每个元素是个 3x4 的矩阵）。
  - **`dim=1`**：第二层包装，里面有 3 个元素（每行）。
  - **`dim=2`**（或者 **`dim=-1`**）：最后一层，里面有 4 个元素。

  **负数索引**：`-1` 代表最后一维，`-2` 代表倒数第二维。 `unsqueeze(-2)` 是在倒数第二层“塞入”一个新维度的原因。

- **==广播机制== (Broadcasting)**：不同形状怎么相加？

  如果两个张量的形状不一致，系统会尝试把较小的张量“拉伸”成较大张量的形状。

  > **例 1**：一个矩阵 `[3, 3]` + 一个数字 `5`。 数字 `5` 会被自动“克隆”成一个 `[3, 3]` 的全是 5 的矩阵，然后对应位置相加。
  >
  > **例 2**：一个 `[3, 1]` 的列向量 + 一个 `[1, 3]` 的行向量。 它们会互相“拉伸”，最终生成一个 `[3, 3]` 的矩阵。

  为什么能相加？ 只要维度从后往前看，要么**相等**，要么**其中一个是 1**，就能广播。

- **==切片== (Slicing)**`[ : ]`：**`[开始 : 结束 : 步长]`**：左闭右开，不包含结束位置

  - **基础切片**：假设 `a = [0, 1, 2, 3, 4, 5]`

    - `a[1:4]` $\rightarrow$ `[1, 2, 3]`（索引 1 - 3）
    - `a[:3]` $\rightarrow$ `[0, 1, 2]`（索引 0 - 2）
    - `a[:-1]` $\rightarrow$ `[0, 1, 2, 3, 4]`（索引 0 - 最后一个之前，即**去掉最后一个**）
    - `a[::2]` $\rightarrow$ `[0, 2, 4]`（每隔 2 个取一个，即步长为 2）

  - **多维切片 `[:, 1:5]`**

    在处理 Transformer 句子时最常用。假设输入 `x` 的形状是 `[Batch, 句子长度]`

    - **`,` (逗号)** 是维度的分隔符
    - **`:`** 在逗号左边，代表**“所有的行（Batch）”**
    - **`1:5`** 在逗号右边，代表**“第 1 到第 4 个单词”**

    `trg[:, :-1]`：保留所有句子（行），但每个句子都去掉最后一个单词。

- **==增加维度==**：`unsqueeze`

  - 假设 `mask` 是 `[10]`（表示 10 个单词的掩码）：

    - `mask.unsqueeze(0)` $\rightarrow$ 变成 `[1, 10]`（变成了一行）。
    - `mask.unsqueeze(1)` $\rightarrow$ 变成 `[10, 1]`（变成了一列）。
    - `mask.unsqueeze(-2)` $\rightarrow$ 在倒数第二维插入。如果原来是 `[Batch, Length]`，执行后变成 `[Batch, 1, Length]`。

  - **为什么要这么做？**：为了触发**广播机制**

    - 多头注意力有 8 个头，形状是 `[Batch, 8, Length, Length]`

      > 注意力矩阵是一个 `[句长,句长]` 的方阵，每个头都独立观察这个方阵，一次处理 `Batch` 个句子。

    - 掩码形状 `[Batch, 1, Length]`

      > 掩码的本质是**“开关”**。它告诉模型：哪些词是真实的，哪些词是补位的 `pad`。
      >
      > 比如句子长度 $L=5$，只有前 3 个词有效，掩码就是 `[1, 1, 1, 0, 0]`。
      >
      > 有 `batch` 个句子，因此它的原始形状是 `[Batch, L]`。
      >
      > 执行 `unsqueeze(-2)`（在倒数第二维加个 1）后，形状变成：**`[Batch, 1, L]`**。

    - **广播**：一个 `[Batch, 1, L]` 的掩码，怎么扣在 `[Batch, 8, L, L]` 的大矩阵上

      - **最后一维 (L)**：都是 $L$，完美对齐。

      - **倒数第二维**：掩码是 **1**，注意力矩阵是 **L**。

        - **广播动作**：掩码会自动把自己**“复制” $L$ 次**。

        - **结果**：现在掩码变成了 `[Batch, 1, L, L]`。这意味着：对于每一行（每一个词），我都应用同样的“开关”规则。

      - **倒数第三维**：掩码是 **1**，注意力矩阵是 **8**（头数）。

        - **广播动作**：掩码再次把自己**“复制” 8 次**。

        - **结果**：掩码变成了 `[Batch, 8, L, L]`。这意味着：所有的 8 个头，都用同样的“开关”规则。

## Torch

- **工具箱**：`torch` 和 `nn`

  - **`torch`**：总库。它定义了什么是 **Tensor（张量）**。
  - **`torch.nn`**：专门用来搭积木。里面全是各种“层”（比如线性层、卷积层）。`nn.Module` 就是所有积木的基座。

- **改变形状**：在 Transformer 里，我们要频繁拆分多头

  - `shape` (查看维度)：`tensor.shape`，查看这个 tenser 的维度。

    > ```py
    > x = torch.randn(64, 12, 300)
    > print(x.shape) # 输出: torch.Size([64, 12, 300])
    > # 意思是有 64 个句子，每句 12 个词，每个词 300 维
    > ```

  - `Linear` (线性层/全连接层)：`nn.Linear(输入维度, 输出维度)`

    > ```py
    > # 定义：输入300维，输出512维
    > layer = nn.Linear(300, 512) 
    > 
    > # 使用：输入一个 [1, 300] 的向量
    > input = torch.randn(1, 300) 
    > output = layer(input) 
    > print(output.shape) # 输出 [1, 512]
    > ```

  - `view` (改变形状)：`tensor.view(新维度1, 新维度2, ...)`

    把张量里的数据按顺序重新排列成你指定的形状，但元素的总个数必须保持不变。

    > 比如 $2 \times 6$ 的矩阵可以变成 $3 \times 4$，但不能变成 $3 \times 5$）。
    >
    > **特殊的 `-1`**：如果你懒得算某个维度的大小，可以写 `-1`，PyTorch 会根据总数自动帮你推算。
    >
    > ```py
    > x = torch.randn(12) # 12 个元素的向量
    > # 变成 3 行 4 列
    > y = x.view(3, 4)  
    > # 变成 2 层，每层 2 行 3 列 (2*2*3 = 12)
    > z = x.view(2, 2, 3)
    > # 自动推算：我想让它变成 6 行，列数你帮我算
    > w = x.view(6, -1) # 自动算出列数是 2
    > ```

  - `permute` (维度置换)：交换维度的顺序，但不改变数据本身。

    `tensor.permute(维度索引0, 维度索引1, ...)`

    > ```py
    > # 原始：[batch, 词数, 头数, 维度] -> [64, 12, 6, 50]
    > x = torch.randn(64, 12, 6, 50)
    > # 换位：把第 1 维(12)和第 2 维(6)对调
    > y = x.permute(0, 2, 1, 3) 
    > print(y.shape) # 输出 [64, 6, 12, 50]
    > ```

  - `contiguous` (内存连续化)：`tensor.contiguous()`，物理修复

    > 当你用 `permute` 交换了维度，数据在电脑内存里变得“乱七八糟”了。有些后续操作（如 `view`）要求数据必须整齐排列，`contiguous` 就像是把书架上的书重新码整齐，方便下一步拿取。
    >
    > ```py
    > x = torch.randn(64, 12, 300)
    > y = x.permute(0, 2, 1) # 换位后，内存变“乱”了
    > # z = y.view(64, 3600) # 这里可能会报错！
    > z = y.contiguous().view(64, 3600) # 这样就安全了
    > ```

- **数学计算**

  - `torch.randn` (生成随机数据)：`torch.randn(维度1, 维度2, ...)`

    > 创建一个指定形状的张量（Tensor），里面的数字是随机生成的，且符合均值为 0、方差为 1 的正态分布（大部分数字在 -1 到 1 之间）。

  - `matmul` (矩阵乘法)：`torch.matmul(A, B)`

    > 遵循线性代数规则：左矩阵的**列**必须等于右矩阵的**行**。
    >
    > ```py
    > A = torch.randn(12, 50)
    > B = torch.randn(50, 10)
    > C = torch.matmul(A, B) 
    > print(C.shape) # [12, 10]
    > ```

  - `softmax` (归一化)：`torch.softmax(tensor, dim=维度)`。

    把一组数变成 **0 到 1 之间的概率**，且总和为 1。

    > 比如分数是 `[10, 5, 1]`，经过 Softmax 可能会变成 `[0.9, 0.09, 0.01]`。这样电脑就知道该“重点关注”第一个词。

  - `masked_fill` (遮盖填充)：`tensor.masked_fill(条件, 填充值)`。

    > 在处理句子时，有些位置是空的（Padding）。我们不希望注意力关注到这些空位。
    >
    > `tensor.masked_fill(mask == 0, -1e10)`：把那些位置的得分强行改成 **负无穷**（-1e10）。这样下一步做 `softmax` 时，负无穷对应的概率就会变成 **0**。
